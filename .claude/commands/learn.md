이번 세션에서 발생한 실수나 피드백을 CLAUDE.md에 강제 규칙으로 기록한다.

## 실행 흐름

1. **실수 파악**: 이번 세션 대화를 돌아보며 반복된 실수, 사용자 정정, 피드백을 최대 5개 이내로 정리
2. **CLAUDE.md 읽기**: `c:\Users\yjisu\Desktop\SCM_WORK\CLAUDE.md` 읽어 기존 강제 규칙 확인
3. **중복 확인**: 이미 있는 규칙이면 스킵
4. **추가 제안**: 새로운 규칙을 아래 형식으로 사용자에게 보여줌
5. **사용자 확인**: "이 규칙을 CLAUDE.md에 추가할까요?" 확인
6. **편집**: 승인 시 CLAUDE.md 하단 "## 학습된 강제 규칙" 섹션에 추가

## 규칙 작성 형식

```markdown
- **[날짜] [규칙 제목]**: [구체적인 금지/필수 행동]. 이유: [왜 이 실수가 발생했나]
```

예시:
```markdown
- **2026-04-07 Airtable 직접 수정 금지**: Airtable 레코드는 API로만 읽기. 직접 수정 시 데이터 정합성 깨짐.
- **2026-04-07 마이그레이션 순서**: 스키마 변경 전 반드시 기존 데이터 백업 확인 후 진행.
```

## 인자 없이 실행 시

이번 세션 전체 대화를 분석해서 반복된 실수를 스스로 찾아 제안한다.

## 인자 있이 실행 시 (`/learn <실수 내용>`)

제공된 내용을 바탕으로 규칙을 즉시 작성하고 CLAUDE.md에 추가한다.
