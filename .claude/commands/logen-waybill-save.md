# 로젠 운송장번호 → Airtable 자동 저장

## 실행 조건
로젠택배 주문등록/출력(단건) 페이지에서 출력 완료 후 실행

---

## 작업 순서

### STEP 1. 현재 페이지에서 데이터 추출

현재 로젠 페이지(logis.ilogen.com)의 테이블에서 다음을 읽어라:
- **운송장번호** (예: 439-7547-9531)
- **물품명** 컬럼에서 **TO번호** 추출 (예: `TO00015544/PNA50745-앱솔브랩` → `TO00015544`)

물품명 파싱 규칙:
- `TO` 또는 `MM` 으로 시작하는 경우: `/` 앞까지가 주문번호 (예: `TO00015544`, `MM00001234`)
- 주문번호가 없는 경우(홍보물/사은품 등): 해당 행은 건너뜀

추출 결과를 다음 구조로 정리:
```
[
  { 운송장번호: "43975479531", 주문번호: "TO00015544" },
  { 운송장번호: "43975479542", 주문번호: "TO00015598" },
  { 운송장번호: "43975479600", 주문번호: "MM00001234" },
  ...
]
```

> 운송장번호는 하이픈(-) 제거한 숫자만 사용

---

### STEP 2. TO번호 기준으로 그룹핑

같은 TO번호에 운송장이 여러 개인 경우 묶어라:
```
[
  { 주문번호: "TO00015544", 운송장번호들: ["43975479531"] },
  { 주문번호: "TO00015598", 운송장번호들: ["43975479542", "43975479553", "43975479564"] },
  { 주문번호: "MM00001234", 운송장번호들: ["43975479600"] },
  ...
]
```

---

### STEP 3. Airtable에서 SC 레코드 조회

Airtable MCP를 사용해서 각 TO번호에 해당하는 SC 레코드를 찾아라.

- **Base**: Sincerely SCM (운영 베이스)
- **Table**: SC (출하확정 테이블)
- **조회 조건**: `배송요청` 필드에 해당 TO번호 또는 MM번호가 포함된 레코드
- **오늘 날짜**(`출하확정일`) 필터 추가로 범위 좁히기

각 SC 레코드의 Record ID를 가져와라.

---

### STEP 4. 운송장번호 업데이트

찾은 SC 레코드에 운송장번호를 저장:

- **필드명**: `운송장번호`
- **여러 개인 경우**: 스페이스로 구분해서 하나의 텍스트 필드에 저장
  - 예: `"43975479542 43975479553 43975479564"`
- 기존에 값이 있으면 덮어쓰지 말고 **사용자에게 먼저 확인** 후 진행

---

### STEP 5. 결과 보고

처리 완료 후 다음 형식으로 결과를 알려줘:

```
✅ 운송장번호 저장 완료

SC00026623 → 43975479531 (TO00015544)
SC00026675 → 43975479542 43975479553 43975479564 (TO00015598)
SC00026686 → 43975479575 (TO00015609)
SC00026782 → 43975479586 (TO00015673)

⚠️ 미매칭: 439-7688-2014 (TO번호 없음 - 홍보물)
```

---

## 에러 처리

| 상황 | 처리 방법 |
|------|----------|
| TO번호 없는 행 | 스킵하고 미매칭 목록에 추가 |
| Airtable에서 SC 못 찾음 | 사용자에게 알리고 수동 확인 요청 |
| 기존 운송장번호 있음 | 덮어쓰기 전 확인 요청 |
| 로젠 페이지 아닌 경우 | "로젠 출력 완료 페이지에서 실행해주세요" 안내 |
