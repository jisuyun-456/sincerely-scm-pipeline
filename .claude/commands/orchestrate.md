# 전문가 오케스트레이션

사용자 요청을 분석하고 적합한 전문가 조합 패턴을 선택하여 실행합니다.

## 패턴 선택 기준

| 요청 키워드 | 패턴 | 에이전트 조합 |
|------------|------|-------------|
| "분석 + 회계 영향" | 병렬 독립 | D1+D2 병렬 → 통합 |
| "프로세스 → 설계 → 일정" | 순차 파이프라인 | D1→D3→D5 |
| "마감 + 검증" | 감사/검증 | D2→D3 |
| "제안서/보고서" | 컨설팅 보고서 | D5+D1+D3 병렬 → D5 통합 |
| "대시보드/풀스택" | 풀스택 개발 | D3+D4+D1/D2 병렬 |

## 실행 규칙

1. 병렬 가능한 에이전트는 반드시 단일 메시지에서 동시 호출
2. 순차 의존 에이전트는 이전 결과를 받은 후 호출
3. 각 에이전트 결과를 취합해 최종 요약 제공
4. INSERT ONLY 원칙: 모든 DB 작업은 불변 원장 INSERT로만 처리
5. 충돌하는 결론 → 사용자에게 선택지 제시
