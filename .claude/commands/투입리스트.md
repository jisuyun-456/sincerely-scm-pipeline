# 자재 투입 리스트 조회 — /투입리스트

베스트원 외부창고 → 에이원지식산업센터 자재 보충 현황을 Airtable에서 조회하여
**파츠별 그룹핑 + 보충수량 계산** 리포트를 출력한다.

---

사용자가 날짜를 지정한 경우 파싱하고, 없으면 기본값 사용.

| 입력 예시 | 적용 범위 |
|-----------|-----------|
| (없음) | 오늘 ~ 오늘+14일 |
| "다음주" | 다음 월요일 ~ 다음 일요일 |

| "4/30까지" / "4월말" | 오늘 ~ 해당 날짜 |
| "긴급" / "급하게" | 오늘 ~ 오늘+7일 |

## Step 3 — 파츠별 그룹핑 & 보충수량 계산

### 그룹핑 키
`출고자재` 필드 앞부분 PT코드 추출 → 동일 파츠 레코드를 묶는다.
예: `"PT1493-오픈박스(스탠다드)(Small)_화이트_100-4 || PNA50803 || 에이원"` → 키: `PT1493-오픈박스(스탠다드)(Small)_화이트_100-4`
(` || ` 첫 번째 구분자 앞까지를 파츠명으로 사용)

### 수량 계산
```
파츠별 총 발주요청수량 = SUM(float(발주요청수량))  — 빈값은 0 처리
에이원재고 = float(출고자재 실물재고수량)          — 동일 파츠 내 첫 번째 값 사용
보충수량 = MAX(0, 총 발주요청수량 - 에이원재고 + 100)
```

### 정렬
그룹 1: 임가공예정일 오름차순 (날짜가 빠른 것 먼저)
그룹 2: 보충수량 내림차순 (긴급한 것 먼저)

**보충수량 > 0인 파츠만 출력한다. 보충수량 = 0인 파츠는 완전히 생략.**

```

생성: YYYY-MM-DD KST

---

**PT{코드}-{파츠명}** — {N}건
| movement_id | 프로젝트 | 임가공예정일 | 발주요청수량 | 상태 |
|---|---|---|---|---|
| MM00XXXXXX | PNA{코드}-{고객사} | M/D (요일) | {수량} | {상태} |
→ 에이원재고: **{N}개** | 총 발주요청: **{N}개** | **보충수량: {N}개** ✅

---

### 생산투입 (신시어리웨일즈) — 보충필요 {N}개 파츠

**PT{코드}-{파츠명}** — {N}건

→ 에이원재고: **{N}개** | 총 발주요청: **{N}개** | **보충수량: {N}개** ✅

---

### 요약
- 조립투입: 보충필요 {N}개 파츠

- **총 보충 필요 파츠: {N}개**
```

## Step 6 — Airtable 보충수량 필드 업데이트 (선택)

> Airtable REST API는 뷰 정렬 설정을 직접 변경할 수 없음.
> 대신 각 레코드에 보충수량 계산값을 기록 → Airtable에서 해당 필드로 정렬하면 동일 효과.

1. movement 테이블(`tblwq7Kj5Y9nVjlOw`) 스키마 조회 — `보충수량_계산` 필드 존재 여부 확인

## Step 6 — PT 수량/오류 분석 (선택)

사용자가 "분석", "오류", "이상값", "재고갭", "중복" 등을 언급하면 추가 실행.

### `/투입리스트 분석`
1. 전체 미완료 레코드 조회 (날짜 제한 없음, 그룹1+그룹2 모두)
2. PT별 집계:
   - 총 발주요청수량 합계
   - 에이원재고 (PT당 첫 번째 값)
   - 재고갭 = 총 발주요청수량 - 에이원재고
   - 임박 임가공예정일 (가장 빠른 날짜)
   - 보충수량 = MAX(0, 재고갭 + 100)
3. 출력: 재고갭 큰 순으로 정렬, 보충수량 > 0인 파츠 전체 표

출력 포맷:
```
## PT 재고갭 분석 — YYYY-MM-DD

| 파츠명 | 총발주 | 에이원재고 | 재고갭 | 보충수량 | 임박날짜 |
|--------|--------|-----------|--------|---------|---------|
| PT1493-... | 500 | 120 | 380 | 480 | 4/22 |
...
보충필요 파츠: {N}개
```

### `/투입리스트 오류`
이상값 감지만 출력:
1. **중복 레코드**: 같은 PT + 프로젝트 + 날짜 조합이 2건 이상인 경우
2. **발주수량 0/음수**: `발주요청수량` ≤ 0 인 레코드
3. **에이원재고 음수**: `에이원재고` < 0 인 파츠 (초과 출고 상태)
4. **에이원재고 불일치**: 동일 PT 코드인데 에이원재고 값이 서로 다른 경우

출력 포맷:
```
## 이상값 감지 — YYYY-MM-DD

### 중복 레코드 ({N}건)
| PT | 프로젝트 | 날짜 | 건수 |

### 발주수량 0/음수 ({N}건)
| movement_id | PT | 발주수량 |

### 에이원재고 음수 ({N}개 파츠)
| PT | 에이원재고 |

### 에이원재고 불일치 ({N}개 파츠)
| PT | 값들 |
```

## 주의 사항

- `발주요청수량`, `출고자재 실물재고수량` 필드는 singleLineText → `float()` 변환 필수 (비어있으면 0)
- `임가공 예정일(+요일)` 파싱: `"M/D (요일)"` → `(month, day)` 추출, 연도는 현재 연도 적용
  - 단, 파싱한 날짜가 오늘보다 2개월 이상 과거이면 내년으로 처리 (연말 넘김 방지)
- 레코드가 많을 경우 `nextCursor` 존재 시 페이지네이션으로 전체 수집 후 처리
- 에이원재고가 음수인 경우 그대로 사용 (이미 초과 출고된 상태 → 보충 더 필요)
