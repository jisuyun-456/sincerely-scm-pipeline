## 1. KPI-1 — 매출액 대비 인건비 비율

### 1.1 KPI 정의

```
KPI-1 = 인건비 합계 / 매출액 합계 × 100 (%)
```

> 목표 방향: 비율 감소 (절대 인건비 감소 X — 회의 의결 D-02 명시)

### 1.2 Issue Tree 분해

```
KPI-1: 매출액 대비 인건비 비율
├── A. 매출액 (분모)
│   ├── A-1. 고객사별 발주 금액 (주문 × 단가)
│   ├── A-2. 발주 빈도 (월간 발주 건수)
│   └── A-3. 객단가 (건당 평균 발주 금액)
└── B. 인건비 (분자)
    ├── B-1. 급여 (기본급 + 수당)
    ├── B-2. 복리후생비
    ├── B-3. 퇴직급여 충당
    └── B-4. AI 대체 경로 (B → 고정→변동 전환 효과)
        ├── B-4a. 현재 수동 작업 비율 (AI 대체 가능 범위)
        └── B-4b. AI 전환 후 인건비 절감 추정
```

### 1.3 K-IFRS + 더존 아마란스10 계정코드 매핑

| 항목 | K-IFRS 분류 | 더존 아마란스10 코드 | 비고 |
|------|------------|-------------------|------|
| 매출액 | IFRS 15 (수익 인식) | 4010X (상품매출) / 4020X (제품매출) | ⚠️ 더존 확인 필요 — 포장재 도소매는 4011 상품매출 추정 |
| 급여 | IAS 19 (종업원급여) | 5011X (급여) | ⚠️ 더존 확인 필요 |
| 복리후생비 | IAS 19 | 5012X (복리후생비) | ⚠️ 더존 확인 필요 |
| 퇴직급여 | IAS 19 | 5013X (퇴직급여) | ⚠️ 더존 확인 필요 |
| 인건비 합계 | — | 5011X + 5012X + 5013X | 직접 집계 가능 |

> **⚠️ 중요**: 신시어리는 포장재 도소매업 추정. 더존 아마란스10 계정 체계는 회사 설정에 따라 코드 세자리 뒤가 다를 수 있음. 실제 계정코드는 더존 회계 담당자 확인 필수.

## 2. KPI-2 — 수수료 계정 활용도

### 2.1 KPI 정의

```
KPI-2a = 수수료 계정 사용 금액 / 인건비 총액 × 100 (%)
         (AI/외주 전환 비율 — 높을수록 좋음)

KPI-2b = 수수료 계정 내 AI 관련 지출 금액 (절대값, 원)
         (AI 투자 추적 지표)
```

> 목표 방향: AI 전환에 따라 수수료 계정 활용 증가, 인건비 절감과 상쇄 추적

### 2.2 Issue Tree 분해

```
KPI-2: 수수료 계정 활용도
├── C. 현재 수수료 계정 현황
│   ├── C-1. 외주가공비 (생산 관련 외주)
│   ├── C-2. 지급수수료 (운영 관련 수수료)
│   └── C-3. AI 관련 SaaS/서비스 비용 (신규)
└── D. AI 전환 비용 추적 경로
    ├── D-1. Claude Code 구독 (AI 도구)
    ├── D-2. Airtable/Railway/Vercel 서비스 비용 (DX 인프라)
    ├── D-3. 외부 AI API 비용 (향후 발생 시)
    └── D-4. 컨설팅·교육비 (AX 관련)
```

### 2.3 K-IFRS + 더존 아마란스10 계정코드 매핑

| 항목 | K-IFRS 분류 | 더존 아마란스10 코드 | 비고 |
|------|------------|-------------------|------|
| 외주가공비 | IAS 2 (재고) / 비용 | 5051X (외주가공비) | ⚠️ 더존 확인 필요 — 포장재 도소매업은 외주가공 적을 수 있음 |
| 지급수수료 | 판관비 | 8051X (지급수수료) | ⚠️ 더존 확인 필요 — 판관비 내 세부 코드 |
| AI SaaS 비용 | 판관비 (소모품 or 수수료) | 8051X 또는 8010X (소모품비) | ⚠️ 더존 확인 필요 — 회사 처리 방침에 따라 상이 |
| 클라우드 인프라 | 판관비 | 8051X 또는 8020X (통신비) | ⚠️ 더존 확인 필요 |

> - 반드시 더존 아마란스10 계정과목표 기준 확인 필요

### 2.4 Baseline / 목표 / 측정주기 / Owner

| 항목 | 값 | 비고 |

| **Baseline KPI-2a** | **미확인** | 더존 계정별 월별 집계 필요. 현재 AI 비용이 별도 계정으로 분류되지 않았을 가능성 높음 |

## 3. KPI-3 — 수평 생산성

### 3.1 KPI 정의

```
KPI-3 = (현재 처리량 또는 자동화율) / (인력 투입 시간 또는 인원) 
       → 2026년 50% 향상 목표
```

> 회의 의결 D-03: "2026 수평 생산성 50% 향상 + 파트별 정렬"  
> "수평 생산성"의 의미: 동일 인원으로 처리 가능한 업무량 증가 (인원 감축 X, 생산량 증대)

### 3.2 Issue Tree 분해

```
KPI-3: 수평 생산성
├── E. 처리량 지표 (분자)
│   ├── E-1. WMS 처리 건수 (입출고 건수/일)
│   ├── E-2. TMS 처리 건수 (운송장 발행/일)
│   ├── E-3. 자동화 작업 비율 (AI/자동 처리 %)
│   └── E-4. 오류율 (재작업 감소 = 생산성 향상)
└── F. 인력 투입 지표 (분모)
    ├── F-1. SCM 담당 인원 (FTE)
    ├── F-2. 인당 처리 건수 (E/F 비율)
    └── F-3. 총 업무 시간 (정규 + 초과근무)
```

### 3.3 파트별 생산성 지표

| 파트 | 현재 측정 가능 지표 | 데이터 위치 | 비고 |
|------|-----------------|-----------|------|
| WMS 입하 | 입하 건수/일 | Airtable wms_movements (type=GR) | ✅ 즉시 측정 가능 |
| WMS 출고 | 출고 건수/일, 피킹건수 | Airtable wms_movements (type=601) | ✅ 즉시 측정 가능 |
| TMS 배송 | 운송장 발행 건수/일 | Airtable tms_shipments | ✅ 즉시 측정 가능 |
| TMS OTIF | OTIF율, 약속납기 | Airtable tms_otif | ✅ 즉시 측정 가능 |
| AutoResearch | 보고서 자동화 건수 | TMS AutoResearch 스크립트 로그 | ✅ 측정 가능 |
| 정산 | 정산 처리 건수/오류율 | FastAPI 정산 harness | ⚠️ 부분 측정 (오류 로그) |
| SAP 시뮬 | VSAP 시뮬레이션 건수 | Supabase vsap_* | ✅ 즉시 측정 가능 |
| 전체 인건비 효율 | 건당 인건비 | 더존 × Airtable 크로스 | ❌ 크로스 집계 필요 |

### 3.4 생산성 Baseline 구축 계획

> **현재 상태**: 처리량 지표는 Airtable에서 측정 가능하나 **인력 투입 시간 데이터 없음**  
> → 건당 처리 시간(time-per-unit)이 없으면 진정한 생산성 비율 계산 불가

**단기 해결 방안 (Proxy 지표)**:
- **생산성 Proxy = Airtable 처리 건수 / SCM 인원 (FTE)**
- FTE: 현재 몇 명인지 확인 후 고정값으로 분모 설정
- 한계: 업무 복잡도 가중치 없음, 시간 측정 부재

**50% 향상 기준 해석**:
- "2026년 말 기준 동일 인원으로 현재 대비 1.5배 처리량"  
- 또는 "현재 업무량을 66.7% 인원으로 처리 가능"  
- → Sprint 4 OKR에서 정의 확정 (회의에서 어느 쪽인지 미결)

### 3.5 Baseline / 목표 / 측정주기 / Owner

| 항목 | 값 | 비고 |
|------|-----|------|
| **Baseline** | **부분 측정 가능** | 처리 건수(Airtable) ✅ / 인력시간 ❌ |
| **Baseline 추정 (Proxy)** | WMS: ~X건/인/일, TMS: ~Y건/인/일 | 실제 수치는 Airtable 쿼리로 즉시 추출 가능 (Sprint 3 시작 전 권장) |
| **목표** | 2026년 말 50% 향상 (회의 의결값) | 정확한 기준 지표 Sprint 4 OKR 확정 |
| **측정주기** | 월 1회 (Airtable 월별 집계) | 자동화 가능 (AutoResearch 확장) |
| **Owner** | scm실장 + TMS AutoResearch 담당 (Claude Code) | |
| **산출 공식 가용성** | ✅ 부분 가능 (Proxy) — Airtable 쿼리로 즉시 추출 | 완전한 인력효율 = 더존 인건비 크로스 필요 |
