# 하네스 엔지니어링 적용 보고서

## 하네스 엔지니어링이란?

Claude Code의 CLAUDE.md 파일을 통해 AI의 작업 흐름을 **구조화·강제·반복**하는 기법.
Anthropic Superpowers 스킬 기반으로, 코드 품질과 일관성을 확보하기 위한 워크플로우 자동화.

---

## 적용 전 (Before)

```markdown
# Claude Code 글로벌 워크플로우 규칙

## 코드 구현 후 필수
- 기능 구현이 끝나면 반드시 superpowers:requesting-code-review skill을 호출해
  코드 리뷰어가 계획 대비 검토하게 할 것
- "완료"를 선언하기 전 superpowers:verification-before-completion으로
  빌드/린트/테스트 검증 실행

## HTML/CSS 디자인 작업
- UI 컴포넌트·스타일 변경 시 frontend-design:frontend-design skill 우선 호출
- 디자인 코드는 실제 렌더링 결과를 시뮬레이션한 뒤 제공

## 버그 대응
- 예상치 못한 동작·에러 발생 시 superpowers:systematic-debugging skill 호출
- 원인 파악 없이 즉시 코드 수정 시도 금지

## 중복 방지
- 위 skill들은 강제 워크플로우이며, 사용자가 명시적으로 생략을 요청할 때만 건너뜀
```

### Before 진단

| 항목 | 상태 |
|------|------|
| 사이클 단계 | 2개만 (review, verify) |
| 구조 | 평면 나열 — 메인 흐름과 특수 경로 구분 없음 |
| 스킵 조건 | 없음 — 사소한 작업에도 전체 강제 |
| 피드백 루프 | 없음 — 리뷰 후 어디로 돌아가는지 불명확 |
| 특수 경로 합류점 | 없음 — 버그/디자인 후 메인 사이클 어디서 합류하는지 모호 |

## 적용 후 (After)

```markdown
# Claude Code 글로벌 워크플로우 규칙

## 메인 사이클 (모든 기능 작업에 순서대로 적용)

### ① 구상 — 작업 시작 전
- 새 기능·컴포넌트·동작 변경 등 창의적 작업 시 superpowers:brainstorming 호출
- 사용자 승인 없이 구현 시작 금지

### ② 계획 — 코드 작성 전
- superpowers:writing-plans 호출하여 구현 계획 작성
- 사용자 승인 후 구현 시작 — 계획 없이 코드 수정 금지

> 사소한 작업은 메인 사이클 전체 스킵:
> 오타 수정, 변수명 변경, 1~2줄 단순 수정은 사이클 없이 바로 수정 가능

### ③ 실행 — 계획대로 구현
- superpowers:executing-plans 호출하여 단계별 실행
- 태스크마다 체크포인트 확인

### ④ 검토 — 구현 완료 후
- superpowers:requesting-code-review 호출하여 계획 대비 검토
- Critical/Important 피드백 → ③에서 수정 후 ④ 재실행
- Minor 피드백 → TODO 기록 후 ⑤로 진행

### ⑤ 검증 — "완료" 선언 전
- superpowers:verification-before-completion 호출하여 빌드/린트/테스트 검증
- 증거 없이 "완료" 선언 금지
- 검증 실패 → ③에서 수정 후 ④부터 재실행

→ 다음 기능이 있으면 ①로 돌아감

## 특수 경로 (메인 사이클 대신 적용)

### 버그 대응 — ① 대신 진입
- 예상치 못한 동작·에러 시 superpowers:systematic-debugging 호출
- 원인 파악 없이 코드 수정 금지
- 수정 완료 후 메인 사이클 ④부터 합류

### 디자인 작업 — ① 직후 분기
- UI·스타일 변경 시 frontend-design:frontend-design 호출
- 렌더링 결과 시뮬레이션 후 코드 제공
- 디자인 확정 후 메인 사이클 ②부터 합류

## 강제 규칙
- 위 워크플로우는 필수이며, 사용자가 명시적으로 생략 요청할 때만 건너뜀
```

---

## Before → After 비교표

| 항목 | Before | After |
|------|--------|-------|
| 사이클 단계 | 2개 (review, verify) | **5개** (구상→계획→실행→검토→검증) |
| 구조 | 평면 나열 | **메인 사이클 + 특수 경로** 분리 |
| 스킵 조건 | 없음 | **사소한 작업 전체 스킵** 정의 |
| 피드백 루프 | 없음 | **④→③ 재실행, ⑤→③ 재실행** 명시 |
| 특수 경로 합류점 | 없음 | **버그→④, 디자인→②** 명시 |
| 동사/톤 | 혼용 ("호출할 것"/"호출해") | **통일** ("호출", "금지") |

```

    │
    ▼

│     아이디어 → 설계 스펙 (사용자 승인) │
└─────────────┬───────────────────────┘
              │
    ┌─────────┴─────────┐
    │  디자인 분기?       │
    │  → frontend-design │
    └─────────┬─────────┘
              │

┌─────────────────────────────────────┐
│  ② 계획  superpowers:writing-plans  │ ← 신규 추가
│     스펙 → 구현 계획 (사용자 승인)    │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────┐
│  ③ 실행  superpowers:executing-plans│
│     계획 → 구현 코드                 │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────┐
│  ④ 검토  superpowers:requesting-code-review │ ← 기존
│     Critical/Important → ③으로 돌아감        │
│     Minor → TODO 기록 후 ⑤로                │
└─────────────┬───────────────────────────────┘
              │
              ▼
┌──────────────────────────────────────────────────────┐
│  ⑤ 검증  superpowers:verification-before-completion  │ ← 기존
│     빌드/린트/테스트 증거 기반 확인                      │

              │
              ▼
```

## 5라운드 반복 개선 이력

| Round | 초점 | 주요 변경 |
|-------|------|----------|
| **R1** | 초안 | brainstorming + writing-plans 섹션 추가, ①~⑤ 번호 순서 배치 |
| **R2** | 구조 리뷰 | 메인/특수 구조 분리, executing-plans 추가, 특수 경로 합류점 명시 |
| **R3** | 엣지케이스 | 스킵 조건 추가, 피드백 루프(③↔④), 검증 실패 루프 정의 |
| **R4** | 워딩 정밀화 | 동사 "호출"로 통일, 장황한 표현 간결화, "TODO 기록" 구체화 |
| **R5** | 시나리오 검증 | 4개 시나리오 시뮬레이션 → 스킵 조건을 "전체 사이클 스킵"으로 수정 |

### Round 5 시나리오 테스트 결과

| 시나리오 | 경로 | 결과 |
|---------|------|------|
| "재고 대시보드 만들어줘" | ①→②→③→④→⑤ | 정상 |
| "로그인 버그 고쳐줘" | debugging→④→⑤ | 정상 |
| "랜딩 페이지 디자인 바꿔줘" | ①→디자인→②→③→④→⑤ | 정상 |
| "변수명 userId→user_id" | 전체 스킵 → 바로 수정 | 정상 |

## 핵심 원리

1. **순서 강제**: ①→②→③→④→⑤ 사이클을 CLAUDE.md에 명문화하여 Claude가 단계를 건너뛸 수 없게 함
2. **피드백 루프**: ④/⑤에서 실패 시 ③으로 되돌아가는 루프로 품질 수렴
3. **특수 경로 합류**: 버그·디자인은 별도 진입점에서 시작하되, 메인 사이클에 합류하여 검토·검증을 보장
4. **스킵 조건**: 사소한 작업의 과도한 오버헤드 방지
5. **증거 기반 완료**: "완료"는 빌드/테스트 통과 증거가 있어야만 선언 가능
