문서형 브리프 시스템 소개

신시어리 SCM 팀이 회의·공유용으로 사용하는 editorial HTML 문서 시스템의 구성과 사용법을 정리한 레퍼런스입니다.

작성일2026.05.18 기준Kit v1.0 용도팀 내부 레퍼런스

I개요

문서형 브리프는 일반 마크다운보다 훨씬 깔끔하게 팀원과 정보를 공유하기 위한 editorial HTML 포맷입니다. 브라우저에서 바로 열리며, 파일 1개로 완결됩니다.

Claude Code에 "문서형으로 만들어줘"라고 하면 doc-brief 에이전트가 자동으로 이 형식의 HTML 파일을 docs/briefs/에 생성합니다.

"문서형으로 만들어줘" 한 마디면 회의 자료 한 장이 완성됩니다. 사용 목적
컴포넌트 수
15종
표·수식·카드·흐름도 등
폰트 수
3종
Newsreader · Pretendard · JetBrains Mono
트리거 키워드
1개
"문서형으로 만들어줘"

파일 구성

docs/kit/ ← 키트 원본 (수정 금지) ├── _template.html ← 새 문서 골격 ├── doc.css ← 공유 스타일시트 └── STYLE_GUIDE.md ← 컴포넌트 15종 가이드 docs/briefs/ ← 생성된 브리프 저장 └── {제목} (문서형).html .claude/agents/ └── doc-brief.md ← 에이전트 (트리거 감지)

개념 카드

01
수식 박스.formula

공식·계산식을 mono 폰트로 강조. FORMULA 라벨 자동 부착.

02
노트 박스.note · .warn · .accent

경고(warn)·강조(accent)·기본 3가지 타입. 상단 보더 색으로 구분.

03
로드맵.roadmap

Iter·단계별 검정 박스 + 우측 설명. 세로 타임라인.

노트 박스 예시

사용 팁
콘텐츠 종류에 맞는 컴포넌트를 선택하면 읽는 사람이 정보 계층을 직관적으로 파악할 수 있습니다.
주의
CSS 변수(--ink, --accent 등)를 새로 추가하지 말 것. 기존 13개 변수 안에서만 디자인하십시오.

흐름도 예시

"문서형으로 만들어줘" → doc-brief 에이전트 실행 → STYLE_GUIDE.md + _template.html 읽기 → 콘텐츠 분석 · 챕터 설계 → docs/briefs/{제목} (문서형).html 저장

III사용법

Claude Code에 아래 중 하나로 말하면 됩니다.

기본 트리거
Primary trigger
"문서형으로 만들어줘" — 가장 짧은 트리거. 내용을 함께 전달하거나 MD 파일을 지목하면 됩니다.
파일 지정
With file
"outputs/MH_report.md 문서형으로 만들어줘" — 특정 파일을 명시하면 해당 파일을 읽어 변환합니다.
저장 경로
Output path
항상 docs/briefs/{제목} (문서형).html에 저장됩니다. 팀원에게 그 파일 1개만 보내면 됩니다.

향후 확장 로드맵

현재
기본 15종 컴포넌트텍스트·표·수식·카드·흐름도·트리 등 회의 자료에 필요한 모든 요소 포함.
다음
회의록 자동 연동SK-08 meeting-analysis 산출 MD → 문서형 HTML 자동 변환 파이프라인.
