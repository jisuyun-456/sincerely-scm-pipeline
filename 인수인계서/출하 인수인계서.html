출하 관리자 교육용 가이드라인

신시어리웨일즈 물류사업본부의 출하 업무를 처음 시작하시거나 매뉴얼로 참고하시는 분들을 위한 가이드입니다. 출하확인서 작성 · 협력사 출하요청 · 배차 스케줄링 · 외주 임가공 이동까지 단계별로 안내합니다.

00·1

이 문서는

신시어리웨일즈 · 출하 부문

이 문서는 신시어리웨일즈 출하 관리자 교육용 가이드라인입니다. 2026년 4월 28일 기준으로 작성되었으며, TMS(Airtable 기반)에서 이루어지는 출하 업무를 다룹니다. 자재관리·임가공이 끝난 굿즈를 고객사·파트너사로 내보내는 과정 전체를 순서대로 정리했습니다.

출하 업무는 출하확인서 작성 · 배차 스케줄링 · 배송 파트너 인계 · 정산의 흐름으로 이어집니다. 이 문서는 그 흐름을 따라 구성되어 있습니다.

"출하"는 단순한 발송이 아니라, 임가공된 굿즈를 정확한 수량으로 정확한 곳에 보내고 그 사실을 시스템에 기록하는 일입니다. — 물류사업본부 실무 인수인계 노트
00·2

출하 업무 흐름

시스템 = TMS · 4단계

출하 업무는 다음 4단계로 흘러갑니다. 각 단계는 별도 탭에서 자세히 다룹니다.

1
출하확인서 작성 — Shipment 테이블에서 품목·수량·외박스 입력 → PDF 생성 (탭 01)
2
협력사 출하요청 메일 — 임가공 협력사(다영) 또는 보관 협력사(베스트원)로 메일 발송 (탭 02)
3
배차 스케줄링 — 배송슬롯·기사님 배정·고고엑스/택배 결정 (탭 03)
4
출하 후 정산·확인 — 미수령 알림 · 외주 임가공 이동 (탭 04~07)

배송 요청 연결 (사전 작업)

Zapier 자동화가 실패한 건만

1·2·3 필드 패턴 — 모든 항목 공통

출하확인서를 만들기 위해서는 반드시 품목 · 수량 · 외박스 수량을 레코드에 기재해야 합니다. 세 항목 모두 동일한 패턴을 따릅니다 — 자동 채워진 1번을 확인 → 2번 직접입력에 옮겨 쓰면 → 3번 최종에 반영.

필드 의미
1번 = 자동 집계값 (lookup, 읽기 전용에 가까움) · 2번 = 관리자가 직접 손대는 입력 칸 · 3번 = 1·2번을 합쳐 출하확인서에 박히는 최종 값. 2번이 비어 있으면 1번이 그대로 3번에 반영됩니다.

출고 종류별 작성 기준

PNA · 샘플 · 재고 · 기타

모든 출고가 같은 방식으로 작성되지 않습니다. 어떤 출고는 임가공팀이 기재해둔 정보가 있고, 어떤 출고는 관리자가 직접 확인해야 합니다.

EASYPNA · 샘플 출고

임가공 팀에서 내용을 기재해두기 때문에 정보가 있습니다.

  • 1번 값을 그대로 확인 → 2번에 옮겨 쓰기
  • 잔여분/띠지 등 추가 항목만 보완
  • 3번에 자동 반영되면 PDF 생성

CHECK재고 · 기타 출고

내용이 작성되지 않은 경우가 있어, 직접 품목 · 수량 · 외박스 수량을 확인 후 작성해야 합니다.

  • 현장 박스 직접 카운트
  • WMS에서 재고 좌표 · PT 번호 확인
  • 2번에 처음부터 입력

02·1

협력사 출하요청 — 두 갈래

PNA건 vs 재고건

협력사 출하요청은 협력사로 "우리가 보낼 자재가 있다"고 알리는 사전 메일 발송 작업입니다. 출하 전날 협력사가 입고 준비를 할 수 있도록, 품목·수량·외박스 정보를 정리해서 전달합니다. 요청은 두 가지로 나뉩니다.

PNA임가공 협력사 (다영기획)

고객사 발주 PNA 프로젝트. 다영기획으로 임가공 자재를 보내는 케이스.

  • view에서 곧장 메일 발송까지 가능
  • 1번 출하일 → 2번 외박스 확인 → 3번 입력 → 4번 발송 버튼
  • 임가공팀이 미리 기재한 정보 활용

MM보관 협력사 (베스트원)

베스트원 재고를 출하하는 케이스. 배차 스케줄링이 먼저입니다.

  • 동일 출하일·동일 베스트원 건 합치기 먼저
  • 출하장소 = 베스트원으로 변경 (필수)
  • 배송 방식·슬롯 설정 후 메일 발송

물류신청 처리

내부 vs 외부 구분

물류신청은 "굿즈를 어디에서 어디로 옮겨달라"는 요청입니다. 신청 주체와 출발/도착 위치에 따라 처리 방식이 달라지므로, 가장 먼저 내부 신청 vs 외부 신청을 판단합니다.

INTERNAL내부 물류신청

회사 내부 부서(임가공팀·MD팀·CX팀)에서 올라오는 요청. 시스템 안에서 처리됩니다.

  • TMS에서 직접 Shipment 생성
  • 출고 보관 장소 = 베스트원 · 다영 등 내부 협력사
  • 요청자에게 시스템 알림으로 회신

EXTERNAL외부 물류신청

고객사·거래처 등 외부에서 들어오는 요청. 메일·메신저로 들어오는 경우가 많아 별도 정리 필요.

  • 요청 내용 → TMS에 수기 입력 (자동화 없음)
  • 견적·정산 항목 별도 체크
  • 처리 결과 메일 회신

고고엑스 배차 (라이트)

7-step 프로세스

고고엑스 라이트 앱·웹에서 배차를 요청합니다. 7단계로 진행하며, 각 단계마다 입력값이 정해져 있습니다.

1
앱/웹 접속
고고엑스 라이트 비즈 계정으로 로그인. 계정 정보는 참고 탭의 자격 정보 섹션 참조.
2
출발지 입력
다영기획 / 베스트원 / 에이원센터 중 출하 장소 선택. 즐겨찾기 사용.
3
도착지 입력
수령인 주소 — TMS 수령인 필드 그대로 복사·붙여넣기.
4
차종/옵션
박스 수량에 맞게 다마스/라보/벤 선택. 외박스 5박스 미만은 다마스로 충분.
5
배송 슬롯
오전·오후·긴급 중 선택. 긴급은 +1만원 비용 추가.
6
메모 작성
아래 메모 레시피 참조. SC 번호·외박스·수령인 연락처 필수.
7
호출 → 송장 입력
배차 완료 후 TMS Shipment에 송장번호·기사 정보 기록.

외주 임가공 이동 준비

SERPA → TMS → 배차

PNA 프로젝트 중 일부는 임가공이 두 단계로 진행됩니다. 예: 다영기획에서 1차 임가공 후 → 베스트원으로 이동 → 베스트원에서 보관·재발송. 이때 "이동 자체"를 출하 건으로 만들어줘야 하는데, 그 절차가 외주 임가공 이동 준비입니다.

전체 흐름

1
스케줄 확인 — 임가공팀 일정에서 이동 예정 PNA 픽업
2
SERPA 데이터 정리 — SERPA에서 PNA별 파츠·수량 raw export
3
운송오더 신청 — TMS 외주 임가공 이동 view에서 일괄 운송오더 생성
4
SC 통합 확인 — 1:1 생성된 SC들이 하나의 SC로 통합되었는지 검증

SC 통합 확인

운송오더 신청 후 자동으로 같은 출하일의 PNA들이 하나의 SC로 묶입니다. 예: 16개의 PNA → SC00015334 하나로.
