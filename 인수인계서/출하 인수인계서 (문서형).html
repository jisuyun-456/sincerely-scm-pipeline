출하 인수인계서

신시어리웨일즈 물류사업본부의 출하 업무를 처음 맡는 관리자분을 위한 서술형 인수인계 문서입니다.

기준일2026.04.28 근무지에이원지식산업센터 대상신규·기존 관리자 시스템TMS (Airtable Base)

I서문

이 문서는 신시어리웨일즈 출하 관리자 교육용 가이드라인입니다. 2026년 4월 28일 기준으로 작성되었으며, TMS(Airtable 기반)에서 이루어지는 출하 업무를 다룹니다. 자재관리·임가공이 끝난 굿즈를 고객사·파트너사로 내보내는 과정 전체를 순서대로 정리했습니다.

출하 업무는 출하확인서 작성 · 배차 스케줄링 · 배송 파트너 인계 · 정산의 흐름으로 이어집니다. 이 문서는 그 흐름을 따라 구성되어 있으며, 마지막에 주의가 필요한 케이스와 매일 확인해야 할 체크리스트를 두었습니다.

"출하"는 단순한 발송이 아니라, 임가공된 굿즈를 정확한 수량으로 정확한 곳에 보내고 그 사실을 시스템에 기록하는 일입니다. 물류사업본부 실무 인수인계 노트

II출하 업무 개요

출하 업무는 모두 TMS(Airtable 베이스)에서 이루어집니다. TMS 베이스 안의 주요 테이블은 다음과 같습니다.

배송요청
고객사·내부 팀에서 들어오는 배송 요청 원장.
Shipment
실제 출하 단위 레코드. 출하확인서 작성·배차·정산이 이 테이블 기준으로 진행됩니다.
배송파트너
퀵·택배·화물 등 외부 파트너사 정보.
Location
출발지·도착지 좌표 마스터.
본 챕터 상세 내용은 추가 인수인계 시 채울 예정입니다.TBD

III배송 요청 연결

배송 요청 테이블에 들어온 TO(배송요청)가 Shipment 테이블에 자동 연결되어야 출하 작업이 시작됩니다. 대부분의 건은 Zapier 자동화를 통해 연결되지만, 간혹 자동화가 실패해 수동 연결이 필요한 경우가 있습니다.

① 미연결 건 찾기 UI에서 보기 ↗

배송요청 테이블 → Shipment 연결 view에서 Shipment 컬럼이 비어 있는 행을 찾습니다. 해당 행의 logistics_PK(TO 번호)를 복사합니다.

② Shipment 테이블에 수동 입력 UI에서 보기 ↗

Shipment 테이블 → 배송요청 연결 view로 이동해 새 행을 만들고 두 필드를 입력합니다.

  1. 1
    배송요청 칸에 복사한 TO00010252를 붙여넣기.
  2. 2
    출하확정일에 출고요청일과 동일한 날짜(예: 2025-04-02)를 입력. 출고 요정일 Rollup (from 배송정보) 필드값과 일치시킵니다.
검증
입력 후 Shipment 테이블 상단으로 돌아가면 신규 SC가 추가되어 있고, 출하장소·임가공 장소가 자동 채워진 것을 확인할 수 있습니다. 이후 배차 스케줄링 단계로 넘어갑니다.
자동화가 안 된 이유
대개 배송요청 항목의 필수 필드(출하장소, 출고요청일 등)가 비어 있거나 형식이 어긋나 Zapier가 트리거를 받지 못한 경우입니다. 수동 연결 후에는 CX팀에 원본 누락 필드 정정을 요청하세요.

IV출하확인서 작성법

출하확인서는 Shipment 테이블의 "출하확인서 작업" view에서 작성합니다. TMS 좌측 사이드바의 관리 그룹 안에 view가 들어 있으며, 이 view에 노출된 레코드들이 금일 출하확인서 작업 대상입니다.

출하확인서가 만들어지는 원리 UI에서 보기 ↗

출하확인서는 make · pyt 스크립트 · github로 출력합니다. 출력 전에는 반드시 품목 · 수량 · 외박스 수량을 레코드에 기재해 두어야 합니다. 세 항목 모두 동일한 패턴을 따릅니다 — 자동 채워진 1번 필드의 값을 2번 직접입력 필드에 덮어 쓰면, 3번 최종 필드에 반영됩니다.

필드 패턴
모든 항목이 1 · 2 · 3 세 칸 구조로 되어 있습니다. 1번은 자동 집계값(읽기 전용에 가까움), 2번이 관리자가 직접 손대는 입력 칸, 3번은 1·2번을 합쳐 출하확인서에 실제로 박히는 최종 값입니다. 2번이 비어 있으면 1번 값이 그대로 3번에 반영됩니다.

V협력사 출하요청 (PNA)

협력사 출하요청은 협력사로 "우리가 보낼 자재가 있다"고 알리는 사전 메일 발송 작업입니다. 출하 전날 협력사가 입고 준비를 할 수 있도록, 품목·수량·외박스 정보를 정리해서 전달합니다. 요청은 두 가지 종류로 나뉩니다.

임가공 협력사
다영기획 · PNA건. 임가공을 다영기획으로 보내는 케이스. 본 챕터(IV)에서 다룹니다.
보관 협력사
베스트원 · MM 재고 이동신청건. 베스트원 재고를 출하하는 케이스. 다음 챕터(V)에서 다룹니다.

view 위치 UI에서 보기 ↗

Shipment 테이블의 사이드바에서 협력사 출하 요청 메일 발송 섹션 → (임가공) 출하요청 메일 발송_to임가공협력사 view를 엽니다.

메일 발송 절차 UI에서 보기 ↗

view에는 1번 출하일 · 2번 외박스 포장 물품 및 수량 · 3번 출하 품목기입(협력사 이메일 발송용) · 4번 협력사 출하요청 버튼 네 칸이 한 줄로 배치되어 있습니다.

  1. 1번 출하일을 확인합니다. (출하 전날 알림 메일을 협력사에 발송하기 위함)
  2. 2번 외박스 포장 물품 및 수량을 확인합니다. 출하 수량으로 임가공팀에서 미리 기재한 값입니다.
  3. 물품·수량 확인 후 3번 출하 품목기입(협력사 이메일 발송용) 필드값에 내용을 덮어 씁니다.
  4. 3번 작성을 완료하면 4번 협력사 출하요청 버튼을 눌러 메일을 발송합니다.

발송 후 확인
메일 발송이 완료되면 두 곳에서 결과를 확인합니다. (왼쪽) 메일은 [신시어리:출하요청]PNA… 제목으로 협력사 메일함에 도착합니다. (오른쪽) softr 페이지의 신시어리 패키징 출고 리스트에 동일 건이 노출되며, 협력사 담당자가 출하처리 버튼을 눌러 확인 후 출하해줍니다.

VI협력사 출하요청 (재고)

베스트원 재고에서 출하해야 하는 건은 절차가 조금 다릅니다. 배차 스케줄링 단계에서 여러 Shipment를 하나로 합쳐주는 작업이 먼저 들어가고, 그다음 메일을 발송합니다.

배차 스케줄링 — Shipment 합치기 UI에서 보기 ↗

Shipment 테이블 → 배차 스케줄링 view에서 시작합니다. 재고 출고 건은 임가공 장소 or 출고 보관 장소 필드값이 베스트원으로 지정되어 있고, 배송요청 칸에 MM00021163 같은 MM 번호가 들어 있습니다.

왜 합치는가

합쳐주는 이유는 배차 1 : Shipment N 구조이기 때문입니다. 위 view를 보면 동일 베스트원 건은 수령인(주소) · 수령인 · 입하담당자가 모두 동일합니다 — 에이원센터로 들어오고, 받는 사람도 같습니다. 그래서 같은 차로 한 번에 가져오면 됩니다.

  1. 같은 출하확정일·같은 베스트원 출고 건의 MM 재고 신청을 하나로 묶어줍니다.
  2. 출하장소를 베스트원으로 변경합니다. (반드시 — 파트너 라운지인 베스트원 softr에 노출되는 필터 조건이 "출하장소=베스트원"이기 때문)
  3. 배송파트너 뒤에 있는 배송 방식 필드값을 맞춰서 바꿔줍니다 — 퀵(수도권) → 퀵, 택배 → 택배.
  4. 배송슬롯을 설정합니다. (기본 디폴트는 오전)

XI고고엑스 배차 방법

배송 파트너가 고고엑스로 지정된 건은 고고엑스 웹페이지에서 직접 배차합니다. TMS에서 SC 단위로 정리한 정보를 그대로 옮겨 입력하는 작업입니다.

대상 건 식별 UI에서 보기 ↗

Shipment 테이블의 출하 준비 view에서 배송파트너 = 신시어리 (고고엑스)로 그룹된 건이 대상입니다.

배차 7단계 UI에서 보기 ↗

고고엑스 웹사이트(www.gogox.com) 로그인 후 신청 화면에서 다음 순서로 진행합니다.

  1. 1
    차량 종류 선택 — 일반 / 다마스 / 라보 등 SC 품목 부피에 맞는 차량 선택
  2. 2
    배송메모란에 SC code 입력 — 추가 운임 및 메모 박스의 배송메모 필드에 SC00017023처럼 입력 (정산 매칭용 — 반드시 입력)
  3. 3
    출발지 정보 입력 — 출하장소 주소·연락처·층 정보
  4. 4
    도착지 정보 입력 — TMS 배송정보의 수령지 주소·연락처. 예: "서울특별시 중구 필동로 37 (충무로17가) HSBC빌딩 14층 화물엘리베이터 앞"
  5. 5
    물품란 클릭 → 상품 선택 패널에서 자세히 보기 클릭
  6. 6
    물품 특이사항 입력 — 박스 크기(소/중/대), 수량, 메모 입력. 메모는 기사님께 요청드릴 사항을 적어둠.
  7. 7
    일반 운임 확인 → 계산&저장 버튼 클릭 → 고고엑스 배차 완료

배송메모란의 SC code 입력은 절대 누락하지 마세요. 고고엑스 정산 시 SC와 매칭하는 키값으로 쓰입니다. SC가 누락되면 어느 출하 건에 대한 청구인지 확인이 어려워집니다.
차량 종류 가이드
소박스 1~2개 = 다마스 · 중박스 다수 또는 1m 이상 = 라보 · 1톤 이상 부피 = 1톤 트럭. 부피 가늠이 어려울 땐 한 단계 큰 차량을 선택하는 게 안전합니다.

XII외주 임가공 이동 준비

외주 임가공으로 자재를 보내기 전, Barcode 베이스에서 스케줄을 확인하고 SERPA 베이스에서 재고이동 데이터를 정리한 뒤 운송오더를 신청하는 사전 작업입니다. 이 단계를 마쳐야 TMS에 SC가 자동 생성되고 출하 작업이 이어집니다.

의존 관계
Barcode 스케줄 → SERPA 데이터 정리 → 운송오더 신청 → TMS SC 생성 순서로 흘러가며, 운송오더 신청이 끝나야 TMS 출하준비 view에 SC가 떨어집니다.

이동목적(생산산출 · 조립투입 · 재고이동) 별로 일치시키며 다음을 수행합니다.

2-1. 생산산출 ↔ 조립투입 수량 일치

  1. 1
    생산산출(입고)에 입고수량이 채워져 있는지 확인 — 실제 물건 입하가 완료된 것. 생산산출은 생산품 재고만 나타냅니다(재고품 자재 ✖).
  2. 2
    조립투입은 생산품+재고 자재가 임가공에 투입되는 자재. 생산산출 입고수량 = 조립투입 출고수량으로 일치시킵니다.
  3. 3
    불량이 있으면 입하 수량이 아닌 입고 수량으로 입력. 예: 입하 2030 / 입고 201이면 201을 입력 (불량 2개 제외해 최종 201개 입고).
  4. 4
    재고자재 수량은 자재지원님이 미리 입력해주시므로 생산품 자재만 맞추면 됩니다.

③ 운송오더 신청 UI에서 보기 ↗

경로
SERPA → 이동신청 → 외주임가공 운송오더 신청 탭
  1. 1
    + 버튼을 눌러 운송오더 작성. 신청자·출하 요청일·출하장소·입하장소·배송 방식·결제방식 입력 후 운송오더 생성하기 클릭.
  2. 2
    출고내역 · 입고내역 탭에서 생산품 자재 + 재고품 자재 품목 수량에 맞춰 + 버튼으로 이동 레코드 생성. 이동목적 = 재고이동.
  3. 3
    project 필드 — 생산산출 데이터는 project를 넣고, 재고자재 데이터는 project를 넣지 않습니다.
  4. 4
    출고자재 수량 = 입고자재 수량을 다 맞춰야 다음 단계로 넘어갑니다.
  5. 5
    출하장소 · 입하장소 입력 — 예: 출고자재/출하장소 = 에이원지식산업센터, 입고자재/입하장소 = 다영기획.
  6. 6
    order · 이동수량 — 조립투입 order 데이터와 조립투입 출고수량을 넣어줍니다.
  7. 7
    이동 관련 전달사항 to 내부구성원 / to 외부파트너 란에 PNA29543-써모피셔사이언티픽_외주임가공처럼 일관되게 작성 후 운송오더 신청.

예외 케이스

재고자재 · Order 데이터 없음
샘플용 1개씩 빼는 건은 order 데이터가 없으므로 운송오더에도 해당 자재 1개씩은 order를 안 넣어도 됩니다.
출하장소 = 베스트원 + 자재투입현황 = 이동요청
에이원이 아닌 베스트원에서 임가공 협력사로 투입하는 건. 따로 이동신청 운송오더를 만들어 베스트원 → 임가공 협력사 재고 출고 요청을 드립니다.
자재투입현황 = 직입하예정
자재가 다른 부자재 협력사에서 임가공 협력사로 바로 투입되는 경우 — 운송오더를 생성하지 않아도 됩니다.
project 필드 규칙
생산품 자재 = project 연결 ○ · 재고자재 = project 연결 ✖. 이게 뒤집히면 이후 정산·재고 추적이 꼬입니다.

④ SC 통합 확인 (TMS) UI에서 보기 ↗

운송오더 신청이 완료되면 TMS에서 자동으로 SC 통합이 이루어집니다.

  1. 1
    TMS → 배송요청 테이블 → 외주 임가공 연결 view로 들어가서 이동 관련 전달사항으로 그룹된 행들 확인.
  2. 2
    SC 코드 번호들이 1:1로 다 생성되어서 하나의 SC 코드로 통일되어 있는지 검증 (예: 16건이 모두 SC00015334로).
  3. 3
    Shipment 테이블 → 출하준비 view에서 같은 SC 아래 이동물품(MM) 번호들이 배송유형 필드에 일괄 표시되면 완료.
완료 신호
출하준비 view에서 새 SC(예: SC00015334)의 이동물품들이 한 줄로 정렬되어 보이면 이후는 일반 출하 프로세스(IV 출하확인서 작성법)로 이어집니다.

XV체크리스트 · 용어집

출하확인서 작성 체크리스트 UI에서 보기 ↗

  1. Shipment 테이블 → 출하확인서 작업 view 열기
  2. 레코드별로 출하 품목 1번 → 2번 덮어쓰기 → 3번 확인
  3. 출고 품목 및 수량 1번 → 2번 덮어쓰기 → 3번 확인 (잔여분·샘플 포함 여부 체크)
  4. 외박스 수량 1번 → 2번 덮어쓰기 → 3번 확인
  5. 재고·기타 출고 건은 직접 현장 확인 후 작성
  6. 출고확인서_python_버튼 클릭 → PDF 생성 확인
  7. PDF 출력 → 박스 동봉

협력사 출하요청 체크리스트 (PNA) UI에서 보기 ↗

  1. (임가공) 출하요청 메일 발송 view 열기
  2. 1번 출하일 확인 → 2번 외박스 포장 물품 및 수량 확인
  3. 3번 출하 품목기입 필드에 내용 덮어쓰기
  4. 4번 협력사 출하요청 버튼 클릭 → 메일 발송
  5. 메일함 + softr 신시어리 패키징 출고에서 발송 결과 확인

협력사 출하요청 체크리스트 (재고) UI에서 보기 ↗

  1. 배차 스케줄링 view에서 동일 출하일·동일 베스트원 건 묶기
  2. 출하장소 → 베스트원 변경 (반드시)
  3. 배송 방식 (퀵/택배) + 배송 슬롯 (기본 오전) 설정
  4. 베스트원 (재고) 출하요청 메일 발송 view에서 1번 → 2번 입력
  5. 3번 협력사 출하요청 이메일 발송(재고요청건) 버튼 클릭
  6. 메일함 + softr 신시어리 출하 스케줄에서 발송 결과 확인
