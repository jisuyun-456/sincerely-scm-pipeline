## 안건별 내용 요약

### 1. TMS TO 수량 불일치 — 승승전기 건
- WMS 출고 수량과 TMS TO(Transport Order) 수량 합산 불일치 발생
- 승승전기 품목에서 티바 3개 분량에 해당하는 TO가 분리되지 않아 재연결 작업 필요
- 현재 "전체 출고 수량만 맞으면 OK" 임시 운영 원칙을 적용 중이나, TO 단위 정합성을 확보해야 KPI(OTIF) 추적이 가능
- 조치 방향: WMS 출고 수량을 TO 3개로 쪼개어 수량 합산이 맞도록 조정 후 각 TO를 올바른 배송 TO에 연결

### 2. 경선님 Claude Code / AI 환경 온보딩
- 윤지수가 구축한 Claude Code 환경(CLAUDE.md 기반 에이전트 하네스)을 문경선 PC에 동일하게 구성
- Airtable 재고 베이스 동기화 및 분석 전용 베이스 접근 권한 부여 필요
- 계정 정리 범위: 문경선, 자영님, 명진님 아이디 일괄 정리
- 이번 주 내 온보딩 완료 목표

### 3. WMS GAP 분석 현황 및 AutoResearch 착수
- ILBS 스탠다드 기준 GAP 분석 테이블 및 필드값 정의 작업은 완료 상태
- 논의된 추가 항목: 로케이션 정보, 공급사 SLA 리드타임, 공급사 등급(A/B/C), Wave-Picking Task 테이블
- AutoResearch는 아직 미실행 상태
- 다음 단계: AutoResearch 실행을 통해 SAP 기준 대비 부족 항목을 정식 도출하고 후속 개선 우선순위 수립

### 4. 피킹리스트/출고확인서 라벨 이동 처리 개선
- 출고확인서 품목·수량·특수 수량 레이아웃 개선 완료 — 스크바이 기준 98 단위 출력 검증 통과
- 현장 반응 매우 긍정적으로 확인됨
- 이슈: TMS/셀포드 연동 시 라벨 1개당 약 20초 대기 시간 발생 (단건 API 호출 구조 기인)
- 성능 이슈는 중기 개선 과제로 분류하고, 현장 적용은 우선 진행

### 5. 인수인계 문서 Airtable UI 전환
- 기존 Google Slides/Spreadsheet 기반 인수인계 문서를 Airtable 인터페이스로 전환 완료
- 각 담당자가 직접 입력하는 구조로 설계되어 운영 편의성 향상 기대
- 구체적 기한은 미정으로 추가 일정 확인 필요

### 6. 모바일 자동화 구상 (중기)

- 현재 미구현 상태이며 중기 과제로 분류
